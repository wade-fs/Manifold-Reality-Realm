一、章節的基本結構（固定骨架）

每一章都必須由四段構成：

1. 觀測（Observation）

角色看到一個「不一致現象」

2. 解釋（Interpretation）

角色用已有理解去解釋它（通常是錯的）

3. 擾動（Perturbation）

世界對這個理解做出反應（強化 / 修正 / 崩壞）

4. 重新穩定（Re-stabilization）

形成新的局部實相

一句話：

每一章 = 一次“理解世界 → 世界改寫你理解”的迴圈

二、章節生成的三種來源（事件種子）

每一章都從以下三種「種子」生成：

Seed A：觀測異常（最常用）

世界出現無法用現有模型解釋的現象

例：
水往上流
人同時出現在兩地
一段歷史被所有人記得但不存在

👉 導致：認知模型崩裂

Seed B：理解衝突（門派 / NPC / 文本）

不同理解模型互相矛盾

例：
三個門派解釋同一現象
古籍與現實衝突
神明說法互相矛盾

👉 導致：實相分裂或融合

Seed C：界面碰撞（跨實相接觸）

不同「界」接觸

例：
妖界與人間重疊
天庭規則干涉村落
地府記憶滲透現世

👉 導致：規則混寫

三、章節生成核心公式（很重要）

每一章本質是：

Reality(n+1) = Stabilize(Interpretation(Observation(n)) + Perturbation)

白話：

人類理解 → 世界回應 → 產生新世界版本

四、角色功能（不是人物，是「理解載體」）

在 FSE 中，角色不是主角，而是：

不同“理解模型”的載體

三種角色類型：
1. 固定理解者（Rigid Interpreter）
堅持單一世界觀
容易崩潰
用來製造衝突
2. 漸進理解者（Adaptive Interpreter）
能修正理解
主線角色通常屬於這類
3. 穿越理解者（Cross-Reality Interpreter）
能同時持有多個版本理解
用來推動世界層升級
五、世界反應規則（非常核心）

當某個理解被提出，世界會做三種反應：

1. 強化（Reinforcement）

理解成功 → 世界變得更“符合該模型”

2. 排斥（Rejection）

理解錯誤 → 現象崩壞 / 人死亡 / 規則失效

3. 分裂（Branching）

多種理解同時成立 → 產生多個實相分支

六、章節節奏模型（100章穩定用）

每 10 章一個「認知收斂單元」

1–3章：觀測建立
世界異常出現
還無法解釋
4–6章：模型衝突
多種解釋並存
門派/古籍出現
7–9章：世界反擊
理解開始影響現實
出現死亡 / 崩壞 /升維
10章：新實相穩定
世界規則被重寫
進入下一層理解
七、事件生成模板（你可以直接用來寫章）

每一章其實可以照這個填：

Chapter Template
1. Observation

發生一個違反現有理解的現象

2. Competing Interpretations
門派A解釋
NPC解釋
古籍解釋
3. Interaction

角色選擇一種理解並行動

4. Reality Feedback
世界反應（修正 / 崩壞 / 分裂）
5. New Stable Reality

新的局部規則成立

八、關鍵設計哲學（FSE核心）

這套系統的本質是：

不是世界在推進劇情，而是理解在推進世界版本
