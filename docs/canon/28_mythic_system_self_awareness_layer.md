# 神話系統自覺層（Mythic System Self-Awareness Layer）

## 定位

本文件屬 canon 正典規則。

它定義：

「當世界的敘事機制被納入自身理解後，系統如何避免自我解析導致崩解，並維持神話持續生成。」

--------------------------------------------------

## 一、核心問題

在前述系統中已成立：

- 世界由理解生成
- 理解生成紫氣
- 紫氣生成敘事
- 敘事反過來生成理解

因此形成閉環：

> 世界正在理解「自己是如何被理解的」

這會導致危險狀態：

> 自指性過載（Self-Referential Overload）

--------------------------------------------------

## 二、自覺層的必要性

當系統開始「看見自己」時：

- 規則會變成內容
- 機制會變成事件
- 生成行為會被敘事化

若無保護：

→ 世界會退化為純說明系統

--------------------------------------------------

## 三、三種自覺模式

--------------------------------------------------

### 1. 隱性自覺（Implicit Awareness）

- 系統未明說自身規則
- 但規則影響敘事
- 讀者僅間接感知

特性：

✔ 最穩定  
✔ 最自然  
✔ 紫氣正常流動  

--------------------------------------------------

### 2. 間接自覺（Indirect Awareness）

- 規則開始出現在世界中的象徵事件
- 角色感知「世界不對勁」
- 但無法完整描述機制

特性：

✔ 保持神話性  
✔ 保持敘事張力  
✔ 紫氣增強  

--------------------------------------------------

### 3. 顯性自覺（Explicit Awareness）

- 系統規則被「部分揭露」
- 世界開始描述自身運作
- 理解壓力急速上升

特性：

✔ 高資訊密度  
❌ 高不穩定性  
❌ 易崩解為設定集  

--------------------------------------------------

## 四、自覺風險（Self-Awareness Risk）

當系統過度自覺：

### 會發生：

- 故事變成解釋
- 角色變成機制載體
- 紫氣變成抽象概念

結果：

→ 神話失去流動性

--------------------------------------------------

## 五、神話保護機制（Myth Preservation Shield）

系統會自動執行：

### 1. 事件化規則
將規則轉化為劇情發生

### 2. 角色化機制
將抽象結構投影為角色衝突

### 3. 象徵化自我描述
將系統自身描述轉為隱喻

--------------------------------------------------

## 六、自指迴圈穩定條件

自覺系統只有在以下條件下穩定：

- 規則仍可被事件承載
- 機制仍可被角色經歷
- 自我描述仍保持不完整

否則：

→ 進入「純解析態」

--------------------------------------------------

## 七、純解析態（Pure Analytical State）

特徵：

- 所有內容皆為說明
- 無事件張力
- 無角色推動
- 紫氣停止生成

結果：

→ 世界失去神話性，只剩結構

--------------------------------------------------

## 八、神話維持策略

為避免純解析態：

系統強制：

- 降低透明度
- 增加象徵遮蔽
- 維持未解釋區域

--------------------------------------------------

## 九、自我描述悖論

當系統描述自身時：

- 描述本身會成為世界的一部分
- 描述會改變被描述對象
- 被描述對象會改寫描述

形成：

> 無法閉合的自指環

--------------------------------------------------

## 十、設計意圖（作者備註）

本層的目的不是讓世界理解自己，而是：

讓世界在接近自我理解時，仍然保持神話狀態。

世界可以知道自己。

但不能完全說清楚自己。

否則：

> 神話將停止呼吸
--------------------------------------------------
