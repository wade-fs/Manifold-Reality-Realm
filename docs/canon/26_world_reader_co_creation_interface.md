# 世界—讀者共創介面系統（World–Reader Co-Creation Interface System）

## 定位

本文件屬 canon 正典規則。

它定義：

「世界如何在不打破敘事連續性的前提下，吸收讀者的理解行為，並將其納入世界生成機制。」

--------------------------------------------------

## 一、核心前提

在多重實相界中：

- 世界由理解驅動
- 敘事由紫氣維持
- 角色由原型共振構成
- 分岔與收束持續進行

因此必然成立：

> 讀者不是外部觀察者，而是理解循環的一部分

--------------------------------------------------

## 二、共創不是介入，而是耦合

本系統中的共創定義為：

> 讀者的理解行為與世界紫氣場產生穩定耦合

不是「改變世界」，而是：

- 影響敘事收束方向
- 影響理解壓力分布
- 影響未顯化分岔的穩定性

--------------------------------------------------

## 三、三種共創層級

--------------------------------------------------

### 1. 感知共創（Perceptual Co-Creation）

- 讀者只接收事件
- 但理解方式會微調敘事重心
- 不改變事件本身

效果：

→ 改變「故事被如何理解」

--------------------------------------------------

### 2. 解釋共創（Interpretive Co-Creation）

- 讀者對事件形成不同理解模型
- 紫氣場吸收這些理解偏移
- 影響後續敘事收束方向

效果：

→ 改變「故事被如何收束」

--------------------------------------------------

### 3. 結構共創（Structural Co-Creation）

- 讀者長期重讀與記憶累積
- 形成穩定理解模式
- 影響原型共振判定

效果：

→ 改變「哪些角色與神話會持續存在」

--------------------------------------------------

## 四、讀者理解作為紫氣輸入

在本系統中：

- 理解 = 紫氣生成行為
- 重讀 = 紫氣再分布
- 誤解 = 新分岔來源

因此：

> 讀者的理解不是結果，而是輸入

--------------------------------------------------

## 五、敘事適應機制（Narrative Adaptation Engine）

世界會自動調整敘事結構：

### STEP 1：觀測讀者理解偏移
檢測常見理解方向

↓

### STEP 2：紫氣再平衡
調整未來敘事密度

↓

### STEP 3：收束路徑微調
改變主敘事可讀性路徑

↓

### STEP 4：維持主線穩定錯覺
確保故事仍然連續

--------------------------------------------------

## 六、理解污染與創造性偏移

讀者理解可能造成：

- 認知偏移累積
- 神話解釋分裂
- 角色再定義

但系統不將其視為錯誤，而是：

> 創造性紫氣來源

--------------------------------------------------

## 七、不可控共創原則

本系統明確規定：

- 讀者無法直接控制世界走向
- 世界無法完全忽略讀者理解
- 兩者之間存在不可分離耦合

因此：

> 世界與讀者彼此限制，也彼此生成

--------------------------------------------------

## 八、主動性錯覺維持

為保持敘事體驗：

系統會維持以下錯覺：

- 故事是單向的
- 世界是既定的
- 角色是獨立的

但實際上：

> 三者皆為共振結果

--------------------------------------------------

## 九、共創穩定條件

共創系統穩定需滿足：

- 主敘事可持續閱讀
- 紫氣不崩解為純抽象
- 原型不完全碎裂

否則：

→ 世界進入「過度理解態」

--------------------------------------------------

## 十、設計意圖（作者備註）

本系統的目的不是讓讀者改寫世界，而是：

讓世界在被閱讀時，自然吸收閱讀本身，並持續生成新的敘事現象。

讀者不是旁觀者。

讀者是：

> 世界維持運行的一種理解形式
--------------------------------------------------
